# Onboarding Processes

Обобщённое описание HR-процессов онбординга в производственной компании (инженерно-монтажный бизнес, 50-100 сотрудников, 2 юр. лица).

## Роли в процессе

| Роль | Что делает |
|---|---|
| HR-менеджер | Дирижёр: ведёт кандидата от оффера до 90-дневного review |
| Бухгалтер по ЗП и кадрам | 1С:ЗУП, Т2, трудовой договор, ознакомление с регламентами |
| Юристы | Трудовой договор, NDA, согласование с Директором юр. департамента |
| Офис-менеджер | Рабочее место, пропуск, канцелярия, приветственная рассылка |
| Линейный руководитель | Установочная встреча, задачи на испытательный срок, знакомство с задачами |
| Системный администратор | Техника, корп. почта, доступы, ИТ-инструктаж |
| Специалист по охране труда | Инструктажи (вводный, пожарный, электробезопасность), медосмотр |
| Делопроизводитель | Приказы, номера приказов |
| Генеральный директор | Подписание документов |

## Этап 1 — Подготовка (до Day 0)

**Триггер:** кандидат принял оффер.

### HR → кандидат
- Приветственное сообщение в мессенджере + просьба прислать скан-копии 11 документов (паспорт, ИНН, СНИЛС, трудовая/СТДР, документы воинского учёта, об образовании, 2-НДФЛ, справка о среднем заработке, свидетельства, №182н, рекомендации)

### HR → внутренние службы (5 email-шаблонов)
1. **ИТ + АХО + линейный руководитель** (+ исп. директор в копию): «К нам выходит новый сотрудник», с ФИО, должностью, датой выхода
2. **Бухгалтерия + юристы**: оффер
3. **Бухгалтерия + юристы** (повтор): документы кандидата для трудоустройства
4. **Делопроизводитель**: запрос номера приказа для должностной инструкции
5. **Офис-менеджер**: копия из резюме нового сотрудника

### Параллельно — подготовка ролей
- **Руководитель**: формулирует индивидуальный план на испытательный срок, заявка сисадмину на технику и доступы
- **Сисадмин**: готовит технику, корп. почту, доступы, программы
- **Офис-менеджер**: рабочее место, канцелярия, пропуск
- **Юристы**: проект трудового договора + NDA, согласование с Директором юр. департамента
- **Бухгалтер**: получает проекты от юристов, распечатывает комплект

## Этап 2 — Первый день (Day 0)

### Утро
- **HR**: встречает у входа, экскурсия, показывает рабочее место, актуальный Welcome book, сопровождает в бухгалтерию, фото → сервер, добавление в чат
- **HR → Специалисту охраны труда**: ФИО, должность, дата рождения, СНИЛС (для инструктажа)
- **Бухгалтер**: вносит в 1С:ЗУП, Т2; проверяет документы; заявления (о приёме, о перечислении ЗП, об авансе — по желанию, о графике — по желанию); ознакомление + подпись под регламентами (внутр. труд. распорядок, оплата, персданные, коммерческая тайна, ДИ, учёт рабочего времени)
- **Офис-менеджер**: именной пропуск (готовится 1-2 недели), карта Тройки (по необходимости), инициирует приветственную рассылку

### День
- **Линейный руководитель**: общие регламенты компании и подразделения, установочная встреча, индивидуальный план (под роспись)
- **Специалист ОТ**: инструктажи (вводный, пожарный, электробезопасность); для монтажников — направление на медосмотр; подпись документов

### Конец дня
- **HR**: обратная связь, доверительные отношения, показывает таблицу контактов

## Этап 3 — Испытательный срок (Day 1 — Day 90)

- Day 3 — HR: «как проходит адаптация?»
- Day 7 — линейный: промежуточная встреча
- Day 14 — HR: пульс-опрос
- Day 30 — HR: 1:1, ревью первого месяца
- Day 90 — HR + линейный: финальный review, решение по окончанию испытательного

## Этап 4 — В штате

- Квартальные 1:1 с HR
- Ежегодный performance review
- Запрос карьерных ожиданий

## Текущие проблемы (без этого инструмента)

1. **Рассинхрон ролей**: HR отправляет email 5 людям, одни начинают действовать сразу, другие видят через сутки
2. **Нет центральной видимости**: невозможно одним взглядом увидеть «что сейчас с Ивановым»
3. **Просрочки теряются**: пропуск заказан — когда будет готов? Ответа нет, всплывает на Day 1 когда новичок пришёл
4. **Ручной welcome**: HR каждый раз копирует сообщение, подставляет имя, отправляет
5. **Новичок теряется**: не знает куда идти, кого спросить; HR не всегда свободен на первый день
6. **Нет ощущения заботы**: важный момент для удержания (первые 90 дней) — но вся забота зависит от настроения HR в конкретный день

## Что решает эта система

1. **Single source of truth** (dashboard) — все онбординги в одном окне
2. **Автоматическое расписание** (бот) — каждая задача улетает в нужный день в нужное время
3. **Welcome-видео от основателя** — каждый новичок получает персональное приветствие с первой минуты
4. **Кнопки в боте** — новичок жмёт «сделано» → событие в dashboard'е → HR видит прогресс в real-time
5. **Вопросы переадресуются** — новичок жмёт «вопрос HR», текст падает HR в TG
6. **Наглядность для собственника** — на одном экране видно «сколько у нас в онбординге / сколько просрочек / кто выходит сегодня»
